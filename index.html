Semana 2: Introducción al HTML.

¿Qué es una página web?

Una página web es un documento de texto con etiquetas. Las etiquetas permiten modificar la presentación del documento, incluir elementos no contenidos en el texto (por ejemplo, imágenes), crear hiperenlaces, añadir significado al texto (todavía de forma limitada), etc. Las páginas web son diseñadas con la intención de ser vistas mediante los programas llamados navegadores web. Estos navegadores web son capaces de interpretar las etiquetas y presentar el documento tal y como lo diseñó su autor.

Etiquetas:

Las etiquetas se escriben entre desigualdades (<h> , </h> ,<p> , </p>, etc.) y suele ir por parejas, abriendo y cerrando textos. La etiqueta de apertura tiene como mínimo el nombre de la misma y la etiqueta para cerrar empieza con el carácter barra ( / ) y contiene únicamente el nombre de la etiqueta.

Además, las etiquetas pueden tener atributos, estos se encargan de especificar algunas caracteristicas de la etiqueta.



Semana 3: Introducción al CSS

¿Qué es el CSS?

CSS (Cascading Style Sheets).
Una hoja de estilo es un documento utilizado para definir como deben lucir los elementos en el HTML, permiten definir colores, tamaños, fuentes, márgenes y más, de manera que el diseño sea consistente y fácil de modificar.

Sintaxis

Una hoja de estilo está compuesta por reglas que contienen selectores y declaraciones. Los selectores identifican qué elementos se verán afectados y las declaraciones definen cómo se verán esos elementos. Las propiedades y sus valores se separan por puntos y comas dentro de bloques de declaración.

Selectores

En CSS, los selectores determinan a qué elementos se aplicarán las propiedades definidas en las reglas. Existen varios tipos de selectores como los de tipo (E), de clase (E.e), de identificador (E#e), de descendientes (E F), de hijos directos (E > F), consecutivos (E + F), y de grupo (E, F). Cada uno selecciona elementos de manera específica según su estructura y jerarquía en el documento HTML.

Especificidad

Cuando múltiples reglas afectan a un mismo elemento, se resuelve usando la especificidad del selector. Esto significa que se aplicará la regla más específica, considerando el número de identificadores, clases y elementos dentro del selector. Si la especificidad es igual, se usa la regla definida más tarde en el código.



Semana 4: Introducción al JavaScript (DOM, HTTP, Y Fetch)

¿Qué es DOM?

DOM permite a los programadores web acceder y manipular los elementos de las páginas HTML a través de objetos llamados nodos.

JSON

es un formato de datos basado en texto que sigue la sintaxis de objeto de JavaScript. Aunque es muy parecido a la sintaxis de objeto literal de JavaScript, puede ser utilizado independientemente de JavaScript, y muchos ambientes de programación poseen la capacidad de leer y generar JSON.

HTTP (Protocolo de transferencia de hipertexto)

Está diseñado para permitir una comunicación entre clientes y servidores.

Funciona como un protocolo de pedido-respuesta entre el cliente y servidor. Un navegador web puede ser el cliente, y la aplicación en una computadora que aloja un sitio web puede ser el servidor.

Método GET

GET es el método HTTP por defecto. Su función es solicitar un recurso específico, por lo tanto las peticiones que utilizan este método solo deben utilizarse para obtener información.

Método POST

El método POST se utiliza para enviar una entidad a un recurso en específico al servidor, causando a menudo un cambio en el estado o efectos secundarios del mismo. Generalmente es utilizado para enviar al servidor información que queremos que el mismo "registre".

Fetch (Comunicación con el servidor)

Fetch es una función integrada en JavaScript que se utiliza para hacer solicitudes HTTP a servidores, obteniendo o enviando datos de manera asíncrona.
